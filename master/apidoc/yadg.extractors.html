yadg.extractors package

yadg.extractors.extract(filetype, path, as_dict=True, orient='tight')

Extract worker function.

Extracts data from provided path, assuming it is the specified filetype. The data is either returned as a dict, where the data tables are dumped by pandas using the appropriate orient keyword, or they are returned with the data tables intact as pd.DataFrames.

Warning

This feature has been introduced in yadg-5.0a2 and is unstable.

Parameters

  • filetype (str) – Specifies the filetype. Can be both yadg-native as well as a MaRDA extractors FileType. Automatically checks for matches using the marda: prefix.

  • path (Path) – A pathlib.Path object pointing to the file to be extracted.

  • as_dict (bool) – A bool flag (default is True) for toggling dict-like export of extracted data (which can be written using json) instead of returning pd.DataFrames.

  • orient (str) – A str argument with which the pd.DataFrames will be exported into dicts.

Returns

ret – A dictionary containing the yadg_metadata and extracted content. Within the content, the metadata contain any metadata present in the file, the values and sigmas contain the nominal values and sigmas of the extracted data, and units contain the units of columns of dimensional quantities.

Return type

dict
